# CLI / Linux / Editors Cheats (RU) — Starter Pack

Этот файл сделан под быстрый поиск по смыслу (что делает), а не по названию команды.

## Как пользоваться (поиск из терминала)
1) Положи файл сюда: `~/cheats/cheats.md`
2) Добавь функцию `cheat` в `~/.zshrc` (или `~/.bashrc`) и ищи так:
- `cheat удалить файл`
- `cheat nvim удалить`
- `cheat nano сохранить`
- `cheat ssh ключ`

## Нотация клавиш
- `Ctrl+X` = зажми **Ctrl** и нажми **X**
- `Alt+X` = **Alt/Option** + X (в документации часто пишут `M-X` / Meta)  
- В Vim/Neovim `Esc` возвращает в Normal mode.

# Права доступа и владение

### chmod
**Ключевые слова:** права chmod permissions 644 755  
**Что делает:** меняет права доступа  
**Команды / клавиши:**
- `chmod 644 file` (rw-r--r--)
- `chmod 755 script.sh` (rwxr-xr-x)
- `chmod -R 755 dir` (рекурсивно)
		chmod u+s file (SUID: запуск с правами владельца; в ls -l будет s вместо x у owner)
	•	chmod g+s dir (SGID на каталог: новые файлы/папки наследуют группу каталога)
	•	chmod +t dir (sticky bit: удалять файлы в общей папке может только владелец файла; как /tmp)
	•	Числа (быстрый вариант):
	•	chmod 4755 file (SUID + 755)
	•	chmod 2775 dir (SGID + 775)
	•	chmod 1777 dir (sticky + 777)

### chown
**Ключевые слова:** владелец chown owner user group  
**Что делает:** меняет владельца/группу  
**Команды / клавиши:**
- `sudo chown user:group file`
- `sudo chown -R user:group dir`
	•	sudo chown :group file (поменять только группу)
	•	sudo chown -R :group dir (рекурсивно только группу)
	•	sudo chgrp group file (то же самое, но отдельной командой)
	•	sudo chgrp -R group dir

### umask
**Ключевые слова:** umask маска прав по умолчанию  
**Что делает:** показывает/задаёт маску прав для новых файлов  
**Команды / клавиши:**
- `umask`
- `umask 022`

# Поиск, фильтрация, обработка текста

### grep
**Ключевые слова:** поиск текста grep find text pattern  
**Что делает:** ищет строки по шаблону  
**Команды / клавиши:**
- `grep -n "ERROR" file`
- `grep -Rni "token" .` (рекурсивно, игнор регистра)

### ripgrep (rg)
**Ключевые слова:** rg ripgrep быстрый поиск текста  
**Что делает:** быстрый grep-поиск (если установлен)  
**Команды / клавиши:**
- `rg "pattern" .`
- `rg -n "TODO" src/`

### find
**Ключевые слова:** найти файл find поиск по имени размеру времени  
**Что делает:** ищет файлы/папки по условиям  
**Команды / клавиши:**
- `find . -name "*.log"`
- `find /var -type f -size +100M`
- `find . -type f -mtime -7` (за 7 дней)

### find: удалить найденное
**Ключевые слова:** find удалить delete -delete rm  
**Что делает:** удаляет то, что нашёл `find`  
**Команды / клавиши:**
- `find . -name "*.tmp" -delete`
- безопаснее: `find . -name "*.tmp" -print` (сначала проверить)

### xargs
**Ключевые слова:** xargs передать в команду много файлов  
**Что делает:** берёт вход и передаёт как аргументы другой команде  
**Команды / клавиши:**
- `find . -name "*.tmp" -print0 | xargs -0 rm -f`

### sed: удалить строки
**Ключевые слова:** sed удалить строку delete line  
**Что делает:** удаляет строки по номеру/шаблону (в потоке)  
**Команды / клавиши:**
- `sed '1,10d' file` (удалить 1–10 строки в выводе)
- `sed '/ERROR/d' file` (удалить строки с ERROR)

### awk: выбрать колонку
**Ключевые слова:** awk колонка поле column  
**Что делает:** обработка по полям/колонкам  
**Команды / клавиши:**
- `awk '{print $1}' file`
- `ps aux | awk '{print $2, $11}'`

### cut
**Ключевые слова:** cut колонка разделитель delimiter  
**Что делает:** вырезает колонки по разделителю  
**Команды / клавиши:**
- `cut -d: -f1 /etc/passwd`

### sort / uniq
**Ключевые слова:** сортировка sort уникальные uniq  
**Что делает:** сортирует и убирает повторы  
**Команды / клавиши:**
- `sort file | uniq`
- `sort file | uniq -c | sort -nr`

# Безопасные “памятки” перед удалением

### Проверить, что будет удалено

**Что делает:** показывает список, не удаляя  
**Команды / клавиши:**
- `find . -name "*.tmp" -print`
- `rm -i file` (спросит)

### Пользователи и группы
Ключевые слова: пользователь создать adduser useradd usermod userdel 
группа groupadd groupdel add to group добавить в группу primary group 
secondary group supplementary groups id getent whoami
Что делает: создаёт пользователей/группы, добавляет пользователя в группы, показывает “кто где состоит”.
Команды / клавиши:
	•	Создать пользователя (часто создаёт home, интерактивно): sudo adduser USER
	•	Создать пользователя (низкоуровнево): sudo useradd USER
	•	Создать группу: sudo groupadd devops
	•	Добавить пользователя в доп. группу (не затирая другие): sudo usermod -aG devops USER
	•	Сделать группу основной (primary): sudo usermod -g devops USER
	•	Проверить кто ты: whoami
	•	Проверить группы пользователя: id USER / groups USER
	•	Посмотреть запись в базе: getent passwd USER / getent group devops
	•	Удалить группу: sudo groupdel devops
	•	Удалить пользователя: sudo userdel USER
	•	Удалить пользователя с home (опаснее, сначала проверь имя): sudo userdel -r USER
Заметки:
	•	После добавления в группу текущая сессия может не увидеть изменения. 
		Обычно помогает перелогин или newgrp devops (на время текущей shell).

Ключевые слова: passwd shadow group /etc/passwd /etc/shadow /etc/group где пароль хранится users groups linux
Что делает: показывает, где в Linux лежат пользователи/группы и почему пароль “не в passwd”.
Команды / клавиши:
	•	Посмотреть пользователя: getent passwd USER / cat /etc/passwd | grep USER
	•	Посмотреть группу: getent group devops / cat /etc/group | grep devops
	•	Проверить shadow (только root): sudo getent shadow USER / sudo cat /etc/shadow | grep USER
Заметки:
	•	/etc/passwd обычно доступен на чтение всем, а хэши паролей лежат в /etc/shadow (и он закрыт для обычных пользователей).
	•	Не “редактируй руками” эти файлы без понимания, проще сломать вход, чем починить.


### Общая папка для группы (SGID-каталог, пример /home/devops)

Ключевые слова: общая папка shared directory group folder setgid sgid chmod 2775 inherit group /home/devops permissions
Что делает: делает каталог общим для группы, чтобы участники могли писать, а группа у новых файлов наследовалась автоматически.
Команды / клавиши:
	•	Создать группу/пользователей: sudo groupadd devops / sudo adduser dev1
	•	Добавить в группу: sudo usermod -aG devops dev1
	•	Создать каталог: sudo mkdir -p /home/devops
	•	Назначить владельца/группу: sudo chown root:devops /home/devops
	•	Права + SGID: sudo chmod 2775 /home/devops
	•	Проверить: ls -ld /home/devops (ищи drwxrwsr-x)
	•	Тест записи: sudo -u dev1 touch /home/devops/testfile
Заметки:
	•	Если упрёшься в Permission denied, сначала проверь ls -ld /home/devops и id dev1, потом уже колдуй.


### Спец. права chmod: SUID / SGID / sticky bit

Ключевые слова: suid sgid sticky bit setuid setgid chmod u+s g+s +t 4755 2775 1777 /tmp
Что делает: включает особые биты прав для файлов/каталогов (часто встречается в системе и общих папках).
Команды / клавиши:
	•	SUID (файл): chmod u+s file / chmod 4755 file
	•	SGID (каталог): chmod g+s dir / chmod 2775 dir
	•	Sticky (каталог): chmod +t dir / chmod 1777 dir
	•	Смотреть, что получилось: ls -l file / ls -ld dir
Заметки:
	•	s в правах файла/каталога это не “красивый значок”, это реально меняет поведение. В общих каталогах обычно нужен именно SGID и иногда sticky.
